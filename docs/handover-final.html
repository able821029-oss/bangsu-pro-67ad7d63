✅ 3. 개발 완료 항목

3-1. 핵심 기능

  • 회원가입 / 로그인 (Supabase Auth)
  • AI 블로그 글쓰기 (Edge Function generate-blog)
  • 쇼츠 영상 생성 (Edge Function generate-shorts + Remotion 렌더러)
  • 나레이션 TTS, BGM 합성, 영상 속도 조절, 이미지 사이즈 조정, 다운로드
  • 게시 일정 관리 (CalendarTab)
  • SEO 점수 분석
  • 플랫폼별 발행 현황 (네이버/인스타/틱톡)
  • 마이페이지 (업체정보, 구독관리, 이용약관, FAQ, 고객문의)
  • Admin 대시보드
  • 현장 도우미 (일당 계산, 날씨 판단, 임금체불 신고)
  • PWA 설치 지원 (홈 화면 추가)
  • 온보딩 슬라이드

3-2. 접근성 (WCAG 2.1 AA)

  • lang="ko" 적용
  • Material Symbols 아이콘에 aria-hidden, 버튼에 한국어 aria-label
  • 포커스 링 전역 스타일 (:focus-visible)
  • <main id="main-content"> 랜드마크 + "본문 바로가기" 스킵 링크
  • 하단 네비에 aria-label="주요 메뉴" + aria-current="page"
  • 차트/게이지에 role="img" + 스크린 리더용 라벨
  • 헤더 "SMS" <h1> 시맨틱 태그

3-3. 코드 품질

  • TypeScript 오류 0건
  • console.log 잔재 0건
  • 더미 데이터 0건
  • Lovable 의존성 완전 제거 (순수 Vite + React)

🔧 5. 운영 가이드

5-1. 코드 수정 → 반영 방법

  1. 개발자가 로컬에서 코드 수정
  2. git add → git commit → git push origin main
  3. Cloudflare Pages 자동 빌드 (1~3분)
  4. sms-app-9p9.pages.dev 자동 반영

5-2. Supabase 관리

  • 대시보드: https://supabase.com/dashboard
  • Edge Function 재배포: supabase functions deploy <name>
  • DB 마이그레이션: Supabase 대시보드 SQL Editor 사용

5-3. 환경 변수

민감 정보는 절대 코드에 하드코딩 금지. 다음 위치에서 관리:

  • Cloudflare Pages: 대시보드 → Settings → Environment variables
  • Supabase Edge Functions: supabase secrets set KEY=value
  • 로컬 개발: .env 파일 (gitignore됨)

⚠️ 7. 주의사항

로컬 빌드 환경 이슈

현재 프로젝트 경로 E:\#앱 개발\SMS앱 개발에 포함된 # 문자 때문에 로컬 npm run build가 실패합니다.

다만 Cloudflare Pages는 원격 서버에서 빌드하므로 이 문제와 무관하게 정상 배포됩니다.

해결 방법: 디렉토리명을 E:\dev\sms-app처럼 # 없는 경로로 변경 권장

Supabase Edge Function 배포 확인 필요

generate-blog, generate-shorts, render-video Edge Function들이 최신 상태로 Supabase에 배포되어 있는지 확인 필요. 코드 수정 시 반드시 supabase functions deploy 실행.
