# SMS 앱 — 종합 진단 및 개선 로드맵 (2026-04-24)

> 대상 커밋: `main` HEAD 기준 · 검토 범위: 프론트엔드(`src/`), Supabase(`supabase/`), 영상 서버(`video-server/`)
> 검토 기준: (1) 코드 구조·아키텍처 (2) 핵심 기능 UX·성능 (3) 보안·안정성 (4) SaaS 관리자 기능

---

## 0. 한 줄 결론

**핵심 5탭 · AI 글쓰기 · 쇼츠 파이프라인 · 영상 서버 5단 방어 계층까지 기본기는 잘 잡혀 있다.**
다만 **SaaS 제품으로 안정 운영하려면 세 가지가 시급**하다.

1. **사용량·비용 측정(`usage_logs`)이 없어서** 누가 얼마를 쓰는지, 한도를 어떻게 강제할지 서버가 모른다.
2. **공개 AI Edge Function(`verify_jwt=false`) 6종에 rate limiting이 없고**, `subscriptions` RLS는 `Allow all`이라 클라이언트에서 플랜·사용량을 자유롭게 조작할 수 있다.
3. **`localStorage`에 `photos[].dataUrl`이 그대로 직렬화되어 저장되고 있어**, 사용자가 글을 수십 개 쌓이면 쿼터 초과로 앱이 로딩 중 흰 화면이 될 위험이 있다.

나머지는 리팩토링·UX 개선 범주로 점진적으로 처리 가능.

### 2.4 SaaS 관리자 기능

현 관리자 페이지는 **설정 편집기** 수준. SaaS 운영에는 아래가 추가되어야 합니다.

| 누락 기능 | 스키마·구현 제안 |
|---|---|
| 사용량/비용 로그 | `usage_logs(user_id, function_name, tokens_input, tokens_output, cost_usd, extra jsonb, created_at)` + 함수별 기록 + 월 집계 뷰 |
| DAU/MAU·가입 퍼널 | `auth.users.created_at` + `usage_logs` 조인. 관리자 대시보드에 차트(Recharts) |
| 결제·환불·이탈 지표 | `subscriptions` 상태(active/cancelled/past_due/refunded) + `payment_events` 테이블 |
| 콘텐츠 모더레이션 | `posts.moderation_status`('ok'/'flagged'/'hidden') + 신고 테이블 `post_reports` |
| 남용 탐지 | `usage_logs` 기반으로 시간당 호출 10건↑, 실패율 50%↑ 사용자를 자동 `flagged` |
| 고객 지원 트레이 | `contact_messages`(있음)에 `assignee_user_id` + 회신 기능 + 이메일 알림 |
| 운영 알림 | video-server 실패율 급증/Edge Function 429 급증 시 Slack webhook (env `ADMIN_SLACK_WEBHOOK`) |

### 적용 후 주의사항

1. **마이그레이션 배포 순서** — Supabase CLI로 올리기 전, `20260419_tighten_rls_and_contact_messages.sql`가 적용되어 `is_admin()` 함수가 존재해야 `20260424_usage_logs.sql`이 정상 적용됨. 이미 적용된 환경이면 문제 없음.
2. **`seo-analyze` 재배포 필수** — `supabase functions deploy seo-analyze --no-verify-jwt` 로 올려야 가드가 반영됨.
3. **CSP 배포 후 점검** — Cloudflare Pages에서 배포 직후 개발자 콘솔의 `Refused to load` CSP 에러 여부 확인. 누락된 도메인이 있으면 `_headers` 보강.
4. **rate limit 튜닝** — `seo-analyze`는 유저 1명이 실제 블로그 개선 사이클에서 30/분을 넘길 이유가 없음. 모니터링 후 10/분까지 내려도 됨.
5. **NaverCallbackPage 제거는 유보** — 아직 `/auth/naver/callback` 구 링크 사용자 가능성으로, 완전 제거 대신 "더 이상 지원하지 않음" 안내 페이지로 교체하는 것을 다음 스프린트 과제로.
